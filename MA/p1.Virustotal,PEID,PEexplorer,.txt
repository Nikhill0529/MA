Lab01-01.exe and Lab01-01.dll
▶  TOOL 1: VirusTotal.com
🔧 Steps to perform:
•	Open browser and go to www.virustotal.com
•	Click "Choose File" and upload Lab01-01.exe
•	Wait for scan results — note how many AV engines detected it (e.g., 44/73)
•	Click "Details" tab → find Compilation Timestamp (date file was compiled)
•	Repeat: upload Lab01-01.dll — note detection count and timestamp
•	If both files have SAME compile timestamp → they are related/associated
✅ Result: Both files compiled around 2010-12-19 16:16:19 — they are related malware

▶  TOOL 2: PEiD
🔧 Steps to perform:
•	Open PEiD application
•	Drag and drop Lab01-01.exe onto PEiD window
•	Read the result line — shows compiler or packer name
•	If it shows "Microsoft Visual C++ 6.0" → file is NOT packed
•	If it shows "UPX" or "FSG" → file IS packed
•	Click "EP Section" button → check section names (UPX0/UPX1 = UPX packed)
•	Repeat for Lab01-01.dll
✅ Result: Both files show Microsoft Visual C++ 6.0 → NOT packed → easy to analyze

▶  TOOL 3: PE Explorer
🔧 Steps to perform:
•	Open PE Explorer → File → Open → select Lab01-01.exe
•	Click "Import Viewer" on left panel → expand each DLL listed
•	KERNEL32.dll → look for: FindFirstFileA, FindNextFileA (searches filesystem)
•	WS2_32.dll (in .dll file) → indicates network functionality
•	Click "Disassembler" → scroll through strings section
•	Look for: kerne132.dll (note: number 1, not letter l) → disguised as kernel32.dll
•	In Lab01-01.dll → find IP address: 127.26.152.13 → network-based indicator
✅ Result: Lab01-01.exe searches filesystem; Lab01-01.dll has network backdoor capability

Lab01-02.exe — UPX Packed File
▶  TOOL 1: PEiD (Detect Packer)
🔧 Steps to perform:
•	Open PEiD → drag Lab01-02.exe onto it
•	First try: Normal scan → shows "Nothing found *"
•	Change scan mode to "Deep Scan" (dropdown next to scan button)
•	Result now shows: UPX packer detected
•	Click "EP Section" → shows sections named UPX0, UPX1, UPX2
•	UPX0 has Virtual Size = 0x4000 but Raw Size = 0 → reserved for unpacked code
✅ Result: File is UPX packed — must unpack to analyze properly


▶  TOOL 2: PE Explorer (Unpack UPX)

•	Now open/reload Lab01-02.exe → it auto-unpacks on load
•	Go to Import Viewer → now see 4 DLLs with real imports:
•	  KERNEL32.DLL → CreateThread (process creation)
•	  ADVAPI32.dll → CreateServiceA (installs as service)
•	  WININET.dll → InternetOpenA + InternetOpenUrlA (connects to URL)
•	Go to Disassembler → find strings: "Malservice", "http://www.malwareanalysisbook.com", "Internet Explorer 8.0"
✅ Result: Malware creates a service called "Malservice" and connects to a URL

Lab03-01.exe — Basic Dynamic Analysis
📌 NOTE: SETUP FIRST: Take VM snapshot → Set network to Host-only → Configure ApateDNS to redirect DNS to 127.0.0.1 → Start iNetSim → Open Netcat listeners on port 80 and 443
▶  TOOL 1: Procmon (Process Monitor)
🔧 Steps to perform:
•	Open Procmon → press Ctrl+X to clear all existing events
•	Add filter: Process Name → is → Lab03-01.exe → Include
•	Add filter: Operation → is → WriteFile → Include
•	Add filter: Operation → is → RegSetValue → Include
•	Run Lab03-01.exe
•	In Procmon, observe: WriteFile to C:\WINDOWS\system32\vmx32to64.exe
•	Observe: RegSetValue at HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\VideoDriver
✅ Result: Malware copies itself as vmx32to64.exe and sets registry key for startup persistence


▶  TOOL 2: Process Explorer
🔧 Steps to perform:
•	Open Process Explorer before running the malware
•	Run Lab03-01.exe → find it in the process list
•	Right-click on process → Properties → Handles tab
•	Look for Mutex named: WinVMX32 → this prevents multiple infections of same machine
•	Go to DLLs tab → see ws2_32.dll and wshtcpip.dll → confirms network capability
✅ Result: Host-based indicators: Mutex WinVMX32, file vmx32to64.exe, registry VideoDriver key


▶  TOOL 3: ApateDNS
🔧 Steps to perform:
•	Open ApateDNS → set "Reply IP" to 127.0.0.1
•	Click "Start DNS Server" button
•	Run the malware → watch DNS requests column
•	Observe: periodic DNS requests to www.practicalmalwareanalysis.com every ~30 seconds
✅ Result: Network-based indicator: malware beacons DNS to www.practicalmalwareanalysis.com every 30s


▶  TOOL 4: Netcat Listener
🔧 Steps to perform:
•	Open Command Prompt 1 → type: nc -lvp 80
•	Open Command Prompt 2 → type: nc -lvp 443
•	Run malware (with ApateDNS redirecting to 127.0.0.1)
•	Netcat on port 443 receives data → shows encrypted/random characters (SSL beacon)
✅ Result: Malware transmits data on port 443 — network-based signature identified
